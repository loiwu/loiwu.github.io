（1）综述

Web和移动应用开发的一般过程：
 
    ---------------     ---------------     ---------------      ----------------   
    |             |     |             |     |             |      |               |
    | 输入/点击链接 | --> | 发送HTTP请求 | --> |    处理请求   | -->  |更新/获取/转换数据|
    |   (Client)  |     |  (Client)   |     |  (Server)   |      |   (Server)    |
    ---------------     ---------------     ---------------      -----------------
                                                                        ↓
                        ---------------     ---------------      ----------------   
                        |             |     |             |      |               |
                        | 呈现HTTP响应 | --> | 接收HTTP响应  | <--  |  返回HTTP响应  |
                        |  (Client)   |     |  (Client)   |      |    (Server)   |
                         ---------------    ---------------      -----------------   

移动应用用原生应用取代浏览器，与普通网站相比，主要区别是：带宽带来的数据传输限制、更小的屏幕、更高效的使用本地存储

四种移动应用的开发方式：

    1.使用编译语言Objective-C/Swift、Java，开发原生的iOS/Android应用
    2.使用脚本语言JavaScript开发，并通过类似[Appcelerator](http://www.appcelerator.com/)的工具，编译成原生应用
    3.通过Responsive Web design、[Bootstrap](http://getbootstrap.com/)、[Foundation](http://foundation.zurb.com/)等，适配小屏幕的移动网站
    4.使用[Sencha Touch](https://www.sencha.com/products/touch/)、[Trigger.io](https://trigger.io/)等，构建包含HTML、CSS、JavaScript的HTML5应用，并使用[PhoneGap](http://phonegap.com/)包装成原生应用
